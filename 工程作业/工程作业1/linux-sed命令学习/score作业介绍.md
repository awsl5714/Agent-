为了帮你巩固 `sed` 的语法和用法，我为你准备了 5 道由浅入深的通关练习题。

这些题目涵盖了实际工作中最高频的使用场景（查看、替换、删除）。建议你可以先在心中或找个文本自己试着写一下，再看下方的“答案与解析”。

---

### 📝 基础准备

假设我们有一个学生成绩单文件 `score.txt`，内容如下：

```text
1: Tom: 85: Pass
2: Jerry: 92: Pass
3: Spike: 58: Fail
4: Tyke: 100: Pass
5: Butch: 45: Fail

```

---

### 🏋️‍♂️ 闯关题目

#### **第一题：查看特定行（初学乍练）**

> **任务**：请写出一条 `sed` 命令，**只打印**出文件 `score.txt` 的**第 3 行**（即 Spike 的那一行）。

#### **第二题：删除整行（渐入佳境）**

> **任务**：因为隐私原因，需要把所有**不及格（包含 `Fail` 关键字）**的同学整行数据从输出中**删除**。请写出对应的 `sed` 命令。

#### **第三题：全局替换（得心应手）**

> **任务**：学校决定把所有的 `Pass` 统一修改为大写的 `EXCELLENT`。请写出替换命令（注意：**不需要**修改原文件，只需在终端打印出修改后的结果）。

#### **第四题：直接修改原文件（熟练运用）**

> **任务**：在第三题的基础上，如果校长要求你**直接把修改结果保存到 `score.txt` 文件中**，让原文件彻底改变，你应该在命令中加入哪个核心选项？

#### **第五题：连续技组合（终极挑战）**

> **任务**：请写出一条命令，**同时完成**两个操作：将 `Tom` 替换为 `Thomas`，并且将 `Jerry` 替换为 `Gerald`。

---

---

### 🔑 答案与详细解析

想好了吗？来对一下答案吧：

#### **第一题答案**

```bash
sed -n '3p' score.txt

```

* **解析**：`3` 是地址，`p` 是动作（print）。别忘了必须要加 `-n` 选项来关闭默认输出，否则 `sed` 会把整个文件打印一遍，然后再多打印一次第三行。

#### **第二题答案**

```bash
sed '/Fail/d' score.txt

```

* **解析**：这里使用了正则表达式定位。`/Fail/` 表示匹配包含 "Fail" 的行，`d` 是删除动作（delete）。这一步不需要 `-n`，因为我们希望 `sed` 默认把其他**没被删除**的行正常打印出来。

#### **第三题答案**

```bash
sed 's/Pass/EXCELLENT/g' score.txt

```

* **解析**：经典的 `s/旧/新/g` 语法。末尾的 `g`（global）表示如果一行里出现多个 "Pass" 统统都替换。

#### **第四题答案**

```bash
sed -i 's/Pass/EXCELLENT/g' score.txt

```

* **解析**：核心在于 **`-i` 选项**（inplace）。不加 `-i` 时，`sed` 只是把处理后的文本“预览”在屏幕上；加上 `-i` 后，屏幕不会有任何输出，修改会直接写入 `score.txt` 文件。

#### **第五题答案**

可以使用以下两种写法的任意一种：

* **方法 A（使用 `-e` 选项连接）**：
```bash
sed -e 's/Tom/Thomas/g' -e 's/Jerry/Gerald/g' score.txt

```


* **方法 B（使用分号 `;` 隔开）**：
```bash
sed 's/Tom/Thomas/g; s/Jerry/Gerald/g' score.txt

```


* **解析**：当需要一次执行多条 `sed` 指令时，方法 A 的 `-e` 代表执行多段表达式（expression）；方法 B 在单引号内用分号 `;` 将不同的动作隔开，这与在命令行中执行多条 Shell 命令的逻辑是一样的。

---

💡 **学习建议**：你可以在自己的 Linux 终端里用 `cat << 'EOF' > score.txt` 命令把这个测试文本创建出来，然后亲自把这 5 条命令敲一遍，看输出是否和预期一致。实践是掌握 `sed` 的最佳途径！
