为了帮你彻底拿下 `awk` 这个大杀器，我同样为你准备了 5 道由浅入深的通关练习题。

`awk` 默认是用**空格**或**制表符（Tab）**来把一行切成一列一列的。我们继续使用之前那个学生成绩单文件 `score.txt`，但这次我们重点观察它的**列结构**。

---

### 📝 基础准备

假设 `score.txt` 内容如下（注意：每列之间是用 **空格** 隔开的）：

```text
1 Tom 85 Pass
2 Jerry 92 Pass
3 Spike 58 Fail
4 Tyke 100 Pass
5 Butch 45 Fail

```

---

### 🏋️‍♂️ `awk` 闯关题目

#### **第一题：提取特定列（小试身手）**

> **任务**：请写出一 条 `awk` 命令，**只打印出所有学生的姓名和成绩**（即第 2 列和第 3 列）。

#### **第二题：条件过滤（略显身手）**

> **任务**：请筛选出**成绩大于或等于 90 分**的所有同学，并打印出他们的整行信息。

#### **第三题：利用行号过滤（融会贯通）**

> **任务**：不使用 `sed` 或 `head`，请纯粹使用 `awk` 的内置变量，**只打印出文件的第 2 行到第 4 行**。

#### **第四题：改变输出格式（得心应手）**

> **任务**：请打印出所有同学的姓名（第 2 列）和状态（第 4 列），但要求输出的格式为：`姓名对应的状态是XX`（例如：`Tom对应的状态是Pass`）。

#### **第五题：改变输入分隔符（终极挑战）**

> **任务**：如果你的朋友给你的文件 `score2.txt` 格式变了，列与列之间不是空格，而是用**冒号 `:**` 隔开的（如下所示）。你该如何修改 `awk` 命令，依然顺利打印出所有人的名字（第 2 列）？
> ```text
> 1:Tom:85:Pass
> 2:Jerry:92:Pass
> 
> ```
> 
> 

---

### 🔑 答案与详细解析

做完之后，来核对一下你的通关秘籍：

#### **第一题答案**

```bash
awk '{print $2, $3}' score.txt

```

* **解析**：`$2` 代表第二列（姓名），`$3` 代表第三列（成绩）。在 `print` 内部，用逗号 `,` 隔开多个变量，`awk` 在输出时会自动帮你加上空格。

#### **第二题答案**

```bash
awk '$3 >= 90' score.txt

```

* **解析**：这是经典的 `awk '模式'` 结构。`$3 >= 90` 是过滤条件（模式）。因为后面没有写 `{print ...}` 动作，`awk` 默认会**打印满足条件的整行内容**（相当于 `{print $0}`）。

#### **第三题答案**

```bash
awk 'NR >= 2 && NR <= 4' score.txt

```

* **解析**：`NR` 是 `awk` 的内置变量，代表**当前行号**（Number of Record）。`&&` 代表逻辑“并且”。由于没有写花括号动作，它会直接打印出第 2、3、4 行。

#### **第四题答案**

```bash
awk '{print $2 "对应的状态是" $4}' score.txt

```

* **解析**：在 `awk` 的 `print` 动作里，如果你想拼接自定义的文本，只需要**用双引号 `""` 把中文括起来**。变量（如 `$2`）和双引号文本之间可以直接连着写，或者用空格、逗号隔开，`awk` 会自动把它们拼接成一句话。

#### **第五题答案**

```bash
awk -F ':' '{print $2}' score2.txt
# 或者连着写
awk -F':' '{print $2}' score2.txt

```

* **解析**：这是实际工作中极常遇到的情况（比如处理系统的 `/etc/passwd` 文件）。核心在于 **`-F` 选项**（Field Separator），它告诉 `awk` 不要再用默认的空格切列了，而是遇到冒号 `:` 就切一刀。

---

### 💡 独家记忆总结

在写 `awk` 时，脑子里一定要有两张地图：

1. **横向地图（行）**：用 `NR`（当前第几行）来控制。
2. **纵向地图（列）**：用 `-F` 指定切刀，用 `$1, $2, $3...` 来拿取切碎后的每一块。
