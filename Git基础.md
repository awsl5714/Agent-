# Git基础

### 1.Git常见命令

1.Git version：查看当前git的版本

2.Git init：git初始化，给当前目录下创建一个`.git`隐藏文件夹，包含所有的版本历史、元数据和对象数据库，只要这个文件夹存在，Git就能追踪所有文件的修改、新增和删除（.git文件经过压缩）

3.Git clone：从远程仓库克隆一个项目到本地（不仅仅是下载最新的代码，而是把该项目完整的开发历史、所有分支、所有的提交记录下载到本地了）（--depth 1 ，下载最近一次提交）（-b 克隆分支）

>https克隆最通用，但是可能需要输入账号密码或者token
>
>SSH克隆：**最方便安全**。需要提前在电脑上生成 SSH 密钥并配置到 GitHub 账号中。配置好后，后续的克隆和推送都**无需再输密码**，非常适合长期开发。
>
>git clone实际上下载的是.git的隐藏文件的压缩包，下载到本地时，会自动执行git checkout，根据最近一次的提交记录，把压缩包的二进制文件还原为实实在在的代码和文件

>## `git init` 与 `git clone` 的区别
>
>很多人刚接触 Git 时容易分不清这两个命令，它们其实代表了两种不同的项目起点：
>
>- **`git init`：** 适用于“从零开始”。你在本地自己新建了一个文件夹，想让 Git 开始管理它，然后你再去关联远程仓库。
>- **`git clone`：** 适用于“中途加入”。远程服务器上已经有一个现成的项目了（可能是你自己的，也可能是别人的），你把它下载到本地开始开发。
>
>> **💡 提示：** > 执行 `git clone` 成功后，Git 会自动帮你建立好本地分支与远程分支的追踪关系（默认命名为 `origin`），你不需要再手动执行 `git init`。直接进入克隆好的文件夹，就可以开始编写代码并进行 `git add` 和 `git commit` 了。

4.git status：查看当前git的状态

>- 未追踪的文件
>- 已修改但是还未暂存的文件
>- 暂存区中的文件

5.git add：把文件从工作区提交到暂存区

`git add.`提交全部文件；git add后面跟着文件名单独提交

6.git commit

>**创建快照：** Git 会把当前暂存区里的文件目录结构和内容打包，固化成一个永久的快照。（进入git）
>
>**生成哈希值（Commit ID）：** Git 会根据提交的内容、作者、时间等信息，通过 SHA-1 算法计算出一个唯一的 40 位字符串（例如 `8f3a5b2...`）。这个哈希值就是这个版本的“身份证号”。
>
>**记录元数据：** 自动记录是谁提交的（Author）、什么时候提交的（Date）以及你的**提交说明（Commit Message）**。
>
>**移动指针：** Git 会把当前分支的指针（比如 `main`）向前推进一步，指向这个刚刚诞生的最新提交。

单行注释提交：git commit -m "想输出的话"

7.git diff：查看详细的代码变动

8.git log：查看提交日志

9.git branch的用法：

>git branch:查看本地所有分支
>
>git branch [分支名]：创建新分支
>
>git branch -d [分支名]：删除分支
>
>git branch -D[分支名]：强制删除分支
>
>git checkout -b 新分支名：创建并切换到该分支
>
>分支在第一次提交的时候才会被创建
>
>----------------------------------------------------------------------------------------------------------
>
>在 Git 的世界里，`feature/add-notes` 是一个典型的**分支命名（Branch Name）**。
>
>它通过使用斜杠 `/` 分隔的方式，清晰地告诉团队两件事：
>
>1. **`feature`（它是干嘛的）**：代表这是一个**功能分支**（Feature Branch）。说明这个分支是专门用来开发新功能的，而不是用来修紧急 Bug（通常叫 `hotfix/`）或做系统发布（通常叫 `release/`）。
>2. **`add-notes`（具体做什么）**：代表这个功能具体的开发内容是“**添加笔记功能**”或“**添加注释**”。
>
>### 💡 为什么大家都要这样命名？
>
>这是一种团队协作中约定俗成的**最佳实践（Naming Convention）**。
>
>在大型项目中，可能会有几十个分支同时存在。如果不规范命名，大家都用 `test`、`my-work`、`update` 这种名字，仓库就会变成一团乱麻。
>
>通过这种“**类型/任务名**”的结构，所有人一眼就能看懂每个分支的职责：

分支的用途：自由的发挥创造，写代码的预览版抢先版！

10.git switch：切换分支

11.git merge：合并分支

>git switch main           # 1. 確保自己站在「接收成果」的主分支上
>git merge feature-login   # 2. 把 feature-login 的代碼拉過來合併

### 2.conflict解决

1. .gitignore(git/ignore,让git忽略的文件)：让git不要追踪这些文件，更不要提交的文件
   >在开发项目时，你的项目文件夹里往往会产生大量**不需要、也不应该**上传到 Git 仓库的文件。例如：
   >
   >- **敏感信息：** 包含你个人密码、数据库连接字符串的配置白名单或密钥文件（如 `.env`）。
   >- **第三方依赖包：** 比如 Python 的 `venv` 虚拟环境、Node.js 的 `node_modules` 文件夹。这些文件体积巨大，别人可以通过配置文件直接下载，没必要塞进你的 Git 仓库。
   >- **编译缓存与临时文件：** 比如 Python 运行产生的 `__pycache__` 文件夹、系统自带的隐藏文件（如 Mac 的 `.DS_Store`）、编译生成的 `.exe` 或 `.o` 文件。
   >- **大文件或日志：** 比如运行日志 `*.log`，或者非常巨大的深度学习模型权重文件。
   >- 写法：
   >- `gitignore` 是一个**纯文本文件**，你只需要在项目的根目录下创建一个名为 `.gitignore` 的文件（注意前面有一个英文句号），然后在里面一行行写下你想忽略的规则即可。
   >- 当你开启一个新项目时，推荐去 **[GitHub 官方的 gitignore 模板仓库](https://github.com/github/gitignore)** 或者 **[gitignore.io](https://www.toptal.com/developers/gitignore)** 这个网站，输入你的编程语言（例如 Python），它会自动帮你生成一份最专业、最全面的 `.gitignore` 文件，你直接复制进项目即可。

2.git restore:

1. 放弃工作区的修改（让代码回到上一次提交/暂存的状态）[相当于ctrl+z]
2. 从暂存区中移出：加上 `--staged` 参数：

>| **命令**                         | **作用（通俗理解）**                               | **影响的区域**               |
>| -------------------------------- | -------------------------------------------------- | ---------------------------- |
>| `git restore --staged <文件>`    | **“我不想提交它了”**（从快递箱里拿出来，放到桌上） | 从上一次提交暂存区拖回暂存区 |
>| `git restore <文件>`             | **“这垃圾代码我不要了”**（撕掉重写，彻底还原）     | 从暂存区拖回工作区           |
>| `git restore --source=HEAD file` | **“我想恢复上一次提交”**                           | 从上一次提交到工作区         |



3.git reset：重置仓库commit提交到某一次版本

>- `--soft`（最温柔：暂存区和工作区不清空，只修改仓库）
>- `--mixed`（默认模式：暂存区清空，工作区不清空）
>- `--hard`（最狠、最危险：彻底抹去，不留痕迹）

4.git revert：做相反的操作，抵消

5.git reflog：调引用日志

执行git reset --hard [哈希值]就可以恢复被误删的代码

6.git stash：代码临时储藏室，把工作区的代码隐藏到后台，等需要使用的时候，用git stash pop恢复

7.git rm 删除本地和工作区中的文件，不更改本地仓库

>#### `git rm --cached <file>`删除工作区，不删除原文件

### 3.推送至远程

1. git remote：建立远程钩子

   >git remote add [别名：通常是origin] 地址：添加一个空白仓库，名字叫origin，和本地项目连起来
   >
   >git remote -v：确认本地目前连到了哪个项目地址
   >
   >git remote set-url [别名] 新地址：修改远程仓库的网址
   >
   >git remote remove [别名]：删除关联的远程仓库

2. git push
